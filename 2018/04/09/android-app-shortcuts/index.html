Android App Shortcuts

发表于 2018-04-09   |   更新于 2018-04-19 | | 阅读次数

如果你的应用的目标是 Android 7.1（API级别25）或更高，则可以在应用中定义快捷方式以适应特定操作。这些快捷方式可以显示在支持的启动器中。Shortcuts（快捷方式）可让你的用户在应用内快速启动常用或推荐的任务。

你可以一次为你的应用发布最多五个快捷方式（包括静态快捷方式和动态快捷方式）。但是，某些启动器应用程序不会显示你为应用程序创建的每个和动态快捷方式。

用户可以创建的应用固定快捷方式的数量没有限制。即使你的应用无法删除已固定的快捷方式，但仍可以禁用它们。

使用静态快捷方式

静态快捷方式应提供应用程序中通用操作的链接，这些操作应在应用程序当前版本的整个生命周期内保持一致。

要创建静态快捷方式，需要以下几个步骤：

  1. 在应用的清单文件（AndroidManifest.xml）中，找到 intent-filter 的 action 设置为 android.intent.action.MAIN , category 设置为 android.intent.category.LAUNCHER的 Activity 。

  2. 为这个 Activity 添加一引用了定义快_捷方式资源文件的 <meta-data> 元素：

  3. 创建一个新的资源文件： res/xml/shortcuts.xml 。

    shortcut 为根元素，其中包含一个 <shortcut> 元素列表。每一个 <shortcut> 元素又包含图标、说明标签和要启动的 Intent 等信息：

    • shortcutId 唯一标识。
    • enabled 是否可用，默认值为 true 。
    • icon shortcut 的图标。
    • shortcutShortLabel 简短标题，建议不超过10个字符。
    • shortcutLongLabel 标题，如果启动器有足够的空间将会显示这个标题，否则会显示短标题 ，建议不超过25个字符。
    • shortcutDisabledMessage 用户点击已固定在桌面的被禁用的快捷方式时显示的提示内容，如果 android:enabled 设置为 true ，此属性无效。
    • intent 点击快捷方式时相应的 Intent ，如果将多个意图与快捷方式相关联，则系统将在资源文件中启动与该快捷方式的最后一个意图相对应的活动，并在后方堆栈中启动其他活动。 在这种情况下，当用户选择快捷方式然后按后退键时，您的应用程序将启动与资源文件中列出的快捷方式倒数第二个意向对应的活动。 这种行为模式会一直重复按下后退按钮，直到用户清除快捷方式创建的后台堆栈。 当用户下一次按下后退按钮时，系统将其导航回启动器。
