使用Hexo搭建GitHub Pages博客

发表于 2016-08-02   |   更新于 2016-08-08   |     |   阅读次数

前言

之前一直想有一个自己的博客，在惠普培训的时候响应老师的号召在CSDN转载了几篇文章算是自己的博客，后来忙着找工作就那样放在那里再也没有看过= =。在看着郭霖的《第一行代码——Android》学习Android的过程中慢慢认识了GitHub，在孔征的”诱惑”下终于决定用GitHub Pages搭建一个自己的博客。

在想要搭建博客到现在搭建完成一共经过了半个多月的时间，期间辞职、找工作，断断续续的看了很多资料，最后终于确定下来用Hexo来搭建自己的博客，主题为NexT（又扯远了。。。），第一篇文章就来记录一下过程吧。

准备

  • GitHub账号
  • Git
  • Node.js
  • Markdown编辑器

(以下操作全部在Windows下完成)

GitHub账号

首先需要一个GitHub账号，然后创建一个名为username.github.io的仓库，username为GitHub账号的用户名，检查仓库设置中的GitHub Pages是否正确，如下图：

Git

下载并安装Git，安装完成后打开Git Bash后分别输入git config --global user.name "用户名"和git config --global user.email "邮箱"来配置用户名和邮箱，再配置一下SSH。

更多GitHub教程详见：从0开始学习 GitHub 系列汇总 。

Node.js

下载并安装Node.js，安装完成后在命令提示符中输入node -v检查，若出现版本则安装正确。

Markdown编辑器

我使用的是Atom，感觉爽到飞起，免费，各种插件，上传图片特别方便(别的我也没用过orz..)

使用Hexo

接下来要进入主题了，是不是很激动呢？

安装Hexo

打开Git Bash，输入

npm install -g hexo-cli

完成！对，你没看错，就是这么简单！

输入hexo -v，出现如下图所示的文字说明安装成功。

安装Git部署所需插件：

& npm install hexo-deployer-git --save

Hexo 3.0或以上版本必须安装hexo server才可在本地预览

$ npm install hexo-server --save

建站

选择一个合适的目录来存放Hexo的各种文件(建议不要放在系统盘，如果重装系统又没有备份就傻眼了)，如目录为D:\Hexo，则在D盘空白处点击右键选择Git Bash Here，执行以下命令：

$ hexo init Hexo
$ cd Hexo
$ npm install

Hexo会在Hexo文件夹中创建需要的文件

配置Hexo

可在Hexo目录下的_config.yml文件中修改大部分配置，如

参数 描述
title 网站标题
subtitle 网站副标题
description 网站描述
author 名字
language 网站使用的语言，如zh-Hans
timezone 网站时区，默认为电脑的时区

部署相关配置：

deploy:
 type: git
 repo: <repository url>
 branch: [branch]
 message: [message]
参数 描述
repo 库（Repository）地址
branch 分支名称
message 自定义提交信息

repository url可在下图位置中找到：

配置过程中可使用yamllint检查语法是否正确。

更多Hexo配置查看Hexo官方文档。

本地查看

执行一下命令出现INFO Hexo is running at http://localhost:4000/. Press Ctrl+C to stop.的提示后可在浏览器输入http://localhost:4000进行查看

$ hexo s

发布

$ hexo g --d

安装NexT主题

下载

继续使用Git Bash在Hexo站点目录下：

$ git clone https://github.com/iissnan/hexo-theme-next themes/next

启用

修改_config.yml文件中的theme字段值为next

theme: next

其他配置官方文档写的非常详细，不再赘述。
