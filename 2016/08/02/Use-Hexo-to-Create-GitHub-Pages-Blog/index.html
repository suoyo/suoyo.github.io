使用Hexo搭建GitHub Pages博客

发表于 2016-08-02   |   更新于 2016-12-29 | | 阅读次数

前言

之前一直想有一个自己的博客，在惠普培训的时候响应老师的号召在CSDN转载了几篇文章算是自己的博客，后来忙着找工作就那样放在那里再也没有看过= =。后来在孔征的”诱惑”下终于决定用GitHub Pages搭建一个自己的博客。

断断续续看了很多资料，最后终于确定下来用Hexo来搭建自己的博客，主题为NexT，赶紧写一篇记录下来搭建的过程。

准备

  • GitHub账号
  • Git
  • Node.js

  • Markdown编辑器

(以下操作全部在Windows下完成)

GitHub账号

首先需要一个GitHub账号，然后创建一个名为username.github.io的仓库，username为GitHub账号的用户名，检查仓库设置中的GitHub Pages是否正确，如下图：

Git

下载并安装Git，安装完成后打开Git Bash后分别输入git config --global user.name "用户名"和git config --global user.email "邮箱"来配置用户名和邮箱，再配置一下SSH。

更多GitHub教程详见：从0开始学习 GitHub 系列汇总 。

Node.js

下载并安装Node.js，安装完成后在命令提示符中输入node -v检查，若出现版本则安装正确。

Markdown编辑器

我使用的是Atom，免费，各种插件，上传图片也特别方便(别的我也没用过orz..)
Markdown语法入门参考：

献给写作者的 Markdown 新手指南

Markdown 编辑器语法指南

使用Hexo

安装Hexo

打开Git Bash，输入

npm install -g hexo-cli

输入hexo -v，出现如下图所示的文字说明安装成功。

安装Git部署所需插件：

& npm install hexo-deployer-git --save

Hexo 3.0或以上版本必须安装hexo server才可在本地预览

$ npm install hexo-server --save

建站

选择一个合适的目录来存放Hexo的各种文件，如目录为D:\Hexo，则在D盘空白处点击右键选择Git Bash Here，执行以下命令：

$ hexo init Hexo
$ cd Hexo
$ npm install

Hexo会在Hexo文件夹中创建需要的文件

配置Hexo

可在Hexo目录下的_config.yml文件中修改大部分配置，如

参数 描述
title 网站标题
subtitle 网站副标题
description 网站描述
author 名字
language 网站使用的语言，如zh-Hans
timezone 网站时区，默认为电脑的时区

部署相关配置：

deploy:
 type: git
 repo: <repository url>
 branch: [branch]
 message: [message]
参数 描述
repo 库（Repository）地址
branch 分支名称
message 自定义提交信息

repository url可在下图位置中找到：

配置过程中可使用yamllint检查语法是否正确。

更多Hexo配置查看Hexo官方文档。

写作

$ hexo new [layout] <title>

可以在命令中指定文章的布局（layout），默认为 post，可以通过修改_config.yml中的default_layout参数来指定默认布局，修改new_post_name参数来改变默认的文件名称，如：

new_post_name: :year-:month-:day-:title.md

创建一篇新文章：

$ hexo new "android studio tips"

本地查看

执行一下命令出现INFO Hexo is running at http://localhost:4000/. Press Ctrl+C to stop.的提示后可在浏览器输入http://localhost:4000进行查看

$ hexo s

设置端口

修改node_modules\hexo-server\index.js文件中的port:值

临时设置端口

在执行时使用-p选项指定其他端口，如下：

$ hexo s -p 5000

发布

$ hexo g --d

安装NexT主题

下载

继续使用Git Bash在Hexo站点目录下：

$ git clone https://github.com/iissnan/hexo-theme-next themes/next

启用

修改_config.yml文件中的theme字段值为next

theme: next

配置

添加文章更新时间

打开themes\next\layout\_macro\post.swig文件，在{% if post.categories and post.categories.length %}前添加

打开\themes\next\languages\zh-Hans.yml文件，在post:下加入(语言不同文件名和添加的内容也不相同)

updated: 更新于

修改主题配置文件themes/next/_config.yml加上

display_updated: true

在文章开头设置更新时间则显示设置的时间，不设置则默认为文件修改时间
